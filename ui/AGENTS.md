# TinyFX — Agent Guide

## What this project is
TinyFX is a hybrid framework: a Rust CLI handles compile-time component expansion
and routing, a TypeScript runtime handles signals, DOM binding, and lifecycle hooks.

## Build pipeline
1. `tinyfx build` scans src/pages/, expands components, writes dist/, generates tinyfx.gen.ts
2. esbuild bundles src/main.ts → dist/js/main.js
3. Styles are copied from src/styles/ → dist/css/

## Key conventions

### Pages
- Pages live in src/pages/ as .html files
- Every page uses a <page> root element with a route attribute
- Example: <page route="/about">...</page>
- Params use colon syntax: <page route="/blog/:slug">

### Components
- Components live in src/components/<Name>/
- Each component has a <Name>.html and an optional <Name>.ts
- Components are used inside pages: <Button variant="primary">Label</Button>
- Components are never routes

### Lib folders
- src/lib/models/   — frontend model classes built from DTOs
- src/lib/dto/      — raw API response types
- src/lib/state/    — signals and global state
- src/lib/services/ — HTTP client wrappers and business logic

### Import aliases — always use these, never use relative paths to lib files
| Alias         | Folder              |
|---------------|---------------------|
| @models/*     | src/lib/models/     |
| @dto/*        | src/lib/dto/        |
| @state/*      | src/lib/state/      |
| @services/*   | src/lib/services/   |

Example:
  import { User } from "@models/user.model";
  import type { UserDto } from "@dto/user.dto";
  import { UserService } from "@services/user.service";

There are no index.ts barrel files. Always import the specific file directly.

## Files you must never edit
- src/tinyfx.gen.ts — auto-generated by tinyfx build, overwritten on every build
- dist/             — entire folder is build output, never edit manually

## generate command
Use tinyfx generate to create new files in the right locations:
  tinyfx generate component Card              → src/components/Card/
  tinyfx generate component Card --dynamic    → src/components/Card/ + Card.ts
  tinyfx generate page   about                → src/pages/about.html + about.ts
  tinyfx generate service AuthService         → src/lib/services/auth-service.service.ts
  tinyfx generate model  Post                 → src/lib/models/post.model.ts
  tinyfx generate dto    Post                 → src/lib/dto/post.dto.ts

Services use factory functions, not classes. Components with --dynamic require tinyfx-dynamic on the HTML root element.
